# Getting Started with Tally

This guide will help you set up Tally and run your first evaluation.

## Installation

```bash
pnpm add @tally-evals/tally
```

## Basic Usage

```typescript
import { createTally, createEvaluator, runAllTargets } from '@tally-evals/tally'
import { createAnswerRelevanceMetric } from '@tally-evals/tally/metrics'

// 1. Define your metrics and evals
const relevance = createAnswerRelevanceMetric({ provider: model })

const evaluator = createEvaluator({
  name: 'Agent Quality',
  evals: [
    defineSingleTurnEval({
      name: 'Answer Relevance',
      metric: relevance,
      verdict: thresholdVerdict(0.7)
    })
  ],
  context: runAllTargets()
})

// 2. Run evaluation
const tally = createTally({ 
  data: conversations, 
  evaluators: [evaluator] 
})
const report = await tally.run()

console.log(report.evalSummaries)
```

## Core Concepts

### 1. Conversations

Tally uses a canonical `Conversation` format for multi-turn exchanges.

```typescript
const conversation = {
  id: "weather-1",
  steps: [
    { 
      input: { role: "user", content: "Hello" }, 
      output: [{ role: "assistant", content: "Hi there!" }] 
    }
  ]
}
```

### 2. Evaluators and Evals

Evals combine measurement logic (Metrics) with pass/fail criteria (Verdicts).

```typescript
const relevanceEval = defineSingleTurnEval({
  name: 'Relevance',
  metric: relevanceMetric,
  verdict: thresholdVerdict(0.7)
})
```

### 3. Storage and Store

Use `@tally-evals/core` for high-level storage operations through `TallyStore`.

```typescript
import { TallyStore } from '@tally-evals/core'

const store = await TallyStore.open({ cwd: process.cwd() })
const conversation = await store.getConversation('conv-123').load()
```
